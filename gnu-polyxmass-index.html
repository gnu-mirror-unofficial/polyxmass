2004-12-08

The versions 0.8.3 of GNU libpolyxmass & GNU polyxmass-bin & GNU polyxmass-common & GNU polyxmass-doc are out. Overall, the changes are to bring a new experimental functionality: the computation of electrical charges for a polymer sequence and also the computation of the isoelectric point for a polymer sequence. For this to happen, the user must edit an XML-formatted ﬁle where pKa data are given for each chemical group (in monomers and modiﬁcations) that might bear an electrical charge. This ﬁle is read by the program (GNU libpolyxmass & GNU polyxmass-bin ) in order to let the user perform the computations.

OVERVIEW

The GNU polyxmass mass spectrometric software is a computer framework for mass spectrometry designed speciﬁcally for (bio-)polymers. This project is modular and involves a number of packages. These packages are reviewed in their respective page that the reader may call by using the links in the menu-table above. While the GNU polyxmass project is not yet ready for production use, it is already useful in a number of ways.

Download its different packages, try it out and report bugs! (see below for the address where to report bugs)

ARCHITECTURE

GNU polyxmass is a collection of packages (containing a library, a binary program and sets of data/conﬁguration ﬁles) that integrate together to provide a mass spectrometry data simulation/analysis framework.

The molecular objects that are dealt with in the GNU polyxmass framework are polymers.

Because the user can deﬁne brand new polymer chemistries easily, the program is not targeted at any polymer chemistry in particular. Indeed, after having deﬁned a polymer chemistry (say, for the biochemist, the protein or the saccharide polymer chemistry), the new polymer chemistry deﬁnition can be used by the GNU polyxmass software suite to perform simulations/analyses on proteinaceous or saccharidic polymer sequences in any way.

The main modules that the user will be using in the GNU polyxmass suite of programs are a library, chemical data, and a GUI (graphical user interface) program. These are described in detail in their respective page (see table-menu above).

HISTORY

GNU polyxmass is a GNU/Linux port of a program that I developed for MS-Windows (massxpert published in Rusconi, F. and Belghazi, M. Bioinformatics 2002 18(4):644-5).

However, GNU polyxmass is not only a port of a pre-existing program, it is a full rewrite that is much more powerful than massxpert. See the Bioinformatics paper here. Indeed, GNU polyxmass can deal with any polymer type, since there is a module that is there to allow the deﬁnition of new polymer types.
