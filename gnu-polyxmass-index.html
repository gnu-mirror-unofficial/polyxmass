OVERVIEW

 The GNU polyxmass mass spectrometric software is a computer framework for mass spectrometry designed specifically for polymers. This project is modular and involves a number of packages. These packages are reviewed in their respective page that the reader may call by using the links in the table above. While the GNU polyxmass project is not yet ready for production use, it is already useful in a number of ways.

Download its different packages, try it out and report bugs!

HISTORY

 GNU polyxmass is a GNU/Linux port of a program that I developed for MS-Windows (massxpert published in Rusconi, F. and Belghazi, M. Bioinformatics 2002 18(4):644-5).

However, GNU polyxmass is not only a port of a pre-existing program, it is a full rewrite that is much more powerful than massxpert. Indeed, GNU polyxmass can deal with any polymer type, since there is a module that is there to allow the definition of new polymer types.

ARCHITECTURE

 GNU polyxmass is a collection of packages (containing a library, a binary program and sets of data/configuration files) that integrate together to provide a mass spectrometry data simulation/analysis framework.

The molecular objects that are dealt with in the GNU polyxmass framework are polymers.

Because the user can define brand new polymer chemistries easily, the program is not targetted at any polymer chemistry in particular. Indeed, after having defined a polymer chemistry (say, for the biochemist, the protein or the saccharide  polymer chemistry), the new polymer chemistry definition can be used by the GNU polyxmass software suite to perform simulations/analyses on proteinaceous or saccharidic polymer sequences in any way.

The main modules that the user will be using in the GNU polyxmass suite of programs are a library, chemical data, and a GUI (graphical user interface) program. These are described in detail in their respective page.
