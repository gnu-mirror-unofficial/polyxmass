HISTORY

 polyxmass is a GNU/Linux port of a program that I developed for MS-Windows (massxpert published in Rusconi, F. and Belghazi,M. Bioinformatics 2002 18(4):644-5).

However, polyxmass is not only a port of a pre-existing program, it is a full rewrite that is much more powerful than massxpert. Indeed, polyxmass can deal with any polymer type, since there is a module that is there to allow the definition of new polymer types.

OVERVIEW

 The polyxmass mass spectrometric software is a computer framework for mass spectrometry specifically for polymers. This project is modular and involves a number of packages. These packages are reviewed in their respective page that the reader may call by using the links on the left.

ARCHITECTURE

 polyxmass is a collection of libraries and programs that integrate together to provide a mass spectrometry data simulation/analysis framework. The molecular objects that are dealt with in the polyxmass framework are polymers.

Because the user can define brand new polymers easily, the program is not targetted at any polymer type in particular. Indeed, after having defined a polymer type (say, for the biochemist, the protein or the saccharide  polymer chemistry), the new polymer definition can be used by the polyxmass suite of programs to perform simulations/analyses on proteinaceous or saccharidic polymer sequences.

The main modules that the user will be using in the polyxmass suite of programs are libraries, chemical data, and GUI (graphical user interface) modules. These are described in detail in their respective page.
